    [issue-188] Implement bounded thread pool and connection limit in Sup…
    228475b 
    …ervisorDaemonEngine

This change addresses a Denial of Service (DoS) vulnerability where an attacker
within the sandbox could cause resource exhaustion by repeatedly connecting
to the supervisor socket, triggering unbounded thread creation.

Changes:
- Added `MAX_CONNECTIONS` limit (default 200) to `SupervisorDaemonEngine`.
- Replaced manual thread creation with a bounded `FixedThreadPool`.
- Implemented connection rejection logic when the limit is reached.
- Added graceful shutdown for the connection executor.
- Added `SupervisorDoSIntegrationTest` to verify the fix.
